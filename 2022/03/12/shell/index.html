shell脚本运行方式

可执行程序

  • 头文件 ·#!/bin/bash,保证我们的脚本默认会使用bash。#! 告诉系统其后路径所指定的程序即是解释此脚本文件的 Shell 程序。
  • 添加权限 chmod +x my.sh
  • 传递参数：$n就代表第几个参数，而$0也就是命令
  • 执行命令 ./my2.sh msg
    • 一定要写成 ./test.sh，而不是 test.sh，运行其它二进制的程序也一样，直接写 test.sh，linux 系统会去 PATH 里寻找有没有叫 test.sh 的，而只有 /bin, /sbin, /usr/bin，/usr/sbin 等在 PATH 里，你的当前目录通常不在 PATH 里，所以写成 test.sh 是会找不到命令的，要用 ./test.sh 告诉系统说，就在当前目录找。

      解释器参数

  • /bin/sh test.sh
  • 不需要在第一行指定解释器信息

shell变量

显式赋值

  • 定义变量时，不加$
  • 变量与赋值=间无空格
  • 不能使用bash关键字。help命令查看
  • readonly只读变量
  • unset命令可以删除变量，不能删除只读变量
  • 可二次赋值

    语句赋值

    使用变量

  • 加$
  • {}识别变量边界;echo "I am good at ${skill}Script"

字符串

  • 单引号：
    • 字符串中的变量无效
    • 不能出现单个的单引号，不能转义
    • 可成对出现，作为字符串拼接
  • 双引号可以出现变量和转义字符

    数组

  • 括号来表示数组，数组元素用”空格”或回车符号分割开
  • array_name=(value0 value1 value2 value3)
  • 可以使用下表单独定义分量，范围无限制，可以不连续
  • 读取${array_name[n]} #[@]获取所有元素

运算符

算数运算符

求和

  • 表达式和运算符之间要有空格
  • 乘号(*)前边必须加反斜杠()才能实现乘法运算
  • MAC 中 shell 的 expr 语法是：$((表达式))，此处表达式中的 “*” 不需要转义符号 “" 。

    赋值

    a=$b

    判断

    [ $a == $b ] [ $a != $b ]
  • 条件表达式要放在方括号之间，前后两边必须要有空格。
  • 同时仍需注意表达式和运算符之间要有空格

关系运算符

  • 只支持数字，不支持字符串，或字符串的值为数字
  • -eq -ne -gt -lt -ge -le

布尔运算符

优先级如下：

  • 非：[ ! false ]
  • 与：[ $a -lt 20 -a $b -gt 100 ]
  • 或：[ $a -lt 20 -o $b -gt 100 ]

逻辑运算符

  • [[ $a -lt 100 && $b -gt 100 ]]
  • [[ $a -lt 100 || $b -gt 100 ]]

字符串运算符

符号 说明 使用
= 检测两个字符串是否相等，相等返回 true。 [ $a = $b ] 返回 false。
!= 检测两个字符串是否不相等，不相等返回 true。 [ $a != $b ] 返回 true。
-z 检测字符串长度是否为0，为0返回 true。 [ -z $a ] 返回 false。
-n 检测字符串长度是否不为 0，不为 0 返回 true。 [ -n “$a” ] 返回 true。
$ 检测字符串是否为空，不为空返回 true。 [ $a ] 返回 true。

echo双引号

  • 输出纯字符串时，字符串的书写形式可以省略引号：echo "aaa"->aaa
  • 可以显示转义字符：echo \"->”

    echo单引号

  • 原样输出字符串，不进行转义或取变量

    显示命令执行结果

  • 使用反引号：

    显示结果可重定向至文件

换行/不换行

  • echo -e "OK! \n" # -e 开启转义

  • echo -e "OK! \c" # -e 开启转义 \c 不换行

    输出变量

    OK #标准输入
    OK It is a test #输出

  • read命令：从标准输入中读取一行,并把输入行的每个字段的值指定给 shell 变量

printf命令

与echo差异

  • 使用 printf 的脚本比使用 echo 移植性好:printf 由 POSIX 标准所定义
  • printf 不会像 echo 自动添加换行符

    格式

    printf format-string [arguments…]

    arguments:参数列表

    format-string:格式控制字符串

  • 双引号单引号效果相同，没有引号也可以输出
  • printf %s abc def 格式只指定了一个参数，但多出的参数仍然会按照该格式输出，format-string 被重用

    a b c
    d e f
    g h i

  • 如果没有 arguments，那么 %s 用NULL代替，%d 用 0 代替

    and 0

    格式控制

  • %s %c %d %f 都是格式替代符，％s 输出一个字符串，％d 整型输出，％c 输出一个字符，％f 输出实数，以小数形式输出。

    ? https://www.runoob.com/linux/linux-shell-printf.html

  • %-10s 指一个宽度为 10 个字符（- 表示左对齐，没有则表示右对齐），任何字符都会被显示在 10 个字符宽的字符内，如果不足则自动以空格填充，超过也会将内容全部显示出来。
  • 小数位数 %-4.2f

循环语句

while

  • Bash let 命令，它用于执行一个或多个表达式，变量计算中不需要加上 $ 来表示变量
  • 读取键盘信息

for

  • ?in 列表可以包含替换、字符串和文件名。

until

case

  • 一旦模式匹配，则执行完匹配模式相应命令后不再继续其他模式。如果无一匹配模式，使用星号 * 捕获该值，再执行后面的命令。
  • 可以匹配变量或常数

break continue

函数

  • 直接fun()定义
  • 参数返回，可以显示加：return 返回，如果不加，将以最后一条命令运行结果，作为返回值。 return后跟数值n(0-255
  • 调用时使用函数名，函数返回值在调用该函数后立刻通过 $? 来获得

    调用 demoFun1 后，没有立即查看 $? 的值，而是先插入了一条别的 echo 命令，最后再查看 $? 的值得到的是 0，也就是上一条 echo 命令的结果，而 demoFun1 的返回值被覆盖了

  • 必须将函数放在脚本开始部分，直至shell解释器首次发现它时，才可以使用
  • 当n>=10时，需要使用${n}来获取参数

重定向

基础

  • 三种流可以同时重定向，举例:
    command < input.txt 1>output.txt 2>err.txt

  • 将 stdout 和 stderr 合并后重定向到 file
    $ command > file 2>&1

  • 管道符号“|”可以连接命令：

    command1 | command2 | command3 | …
    以上内容是将command1的stdout发给command2的stdin，command2的stdout发给command3的stdin，依此类推。

    Here Document

    /dev/null

  • 执行某个命令但不显示结果，将输出重定向到 /dev/null $ command > /dev/null

  • 屏蔽 stdout 和 stderr $ command > /dev/null 2>&1

文件包含

  • 被包含的文件 test1.sh 不需要可执行权限。

其他

多行注释

  • {}：定义成未被调用的函数
  • :<<EOF ... EOF：EOF可替换为!、'

常见符号

  • $# 传递的参数个数

  • $* 一个字符串显示传递的全部参数

  • []是一个判断符号，条件语句要放在[]里面，并且前后两边必须要有空格。

  • 中括号包含的条件表达式的-ne是关系运算符，它们和c语言的比较运算符对应如下。

    -eq ==(equal)
    -ne !=(not equal)
    -gt >(greater than)
    -lt <(less than)
    -ge >=(greater or equal)
    -le <=(less or equal)

条件也可以写true或false。
